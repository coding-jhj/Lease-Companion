# data/AGENTS.md

`data/` 전용 지시서. 루트 [`../AGENTS.md`](../AGENTS.md)를 전제로 하며, 여기서는 데이터 고유 규칙만 정의한다.

## 커밋 금지

- 실제 계약서·개인정보를 커밋하지 않는다.
- 이름·주소·연락처·계좌번호 등 민감정보를 커밋하지 않는다.
- `data/raw` 내부 실제 데이터는 Git에서 제외한다. (`data/raw/README.md`만 예외)

## 샘플 데이터

- `sample/`에는 가상 또는 완전히 비식별화한 데이터만 둔다.
- 실제 자료와 생성 데이터를 명확히 구분한다.

## RAG / 분석 기준 자료

- 공식 법령·표준 주택임대차계약서·공공기관 자료를 우선한다.
- 블로그·카페·커뮤니티 자료를 핵심 근거로 사용하지 않는다.
- RAG 자료마다 **기관명·문서명·URL·발행일 또는 확인일**을 기록한다. (`rag/metadata/`)

## 데이터 구분

- 원본(`raw`)·정제본(`processed`)·청크(`rag/chunks`)·평가 데이터(`evaluation`)를 구분한다.
- 원본 데이터를 직접 덮어쓰지 않는다. 가공은 별도 산출물로 남긴다.

## 규칙 데이터

- 규칙마다 **규칙 ID·적용 단계·입력 필드·조건·결과·근거·버전**을 기록한다. (`docs/data/rule-definition.md`)
