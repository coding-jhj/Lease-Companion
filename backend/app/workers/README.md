# app/workers/

## 책임

비동기·백그라운드 작업 계층. 분석 실행(상용 LLM 구조화 → 규칙 엔진 → RAG → 상용 LLM 생성, 로컬 7B는 선택적 성능비교 실험)처럼
시간이 걸리는 파이프라인을 요청·응답 주기와 분리해 처리한다. 서비스 계층이 작업을
등록하고, 워커가 실행 후 결과를 저장소에 영속화한다.

## 입출력

- 입력: 서비스가 등록한 분석 실행 작업(계약 건·문서·추출값 참조)
- 처리: `ai/` 파이프라인 오케스트레이션, `AnalysisRun` 상태 갱신(분석 중/완료/오류)
- 출력: 판정·근거·질문·행동·체크리스트를 `repositories/`로 저장

## 경계

- 규칙 엔진 판정을 LLM·로컬 모델이 임의로 변경하지 못하게 한다.
- 외부 AI 호출 실패·시간 초과·잘못된 출력을 처리하고 상태에 반영한다.
