# app/services/

## 목표 책임

라우트와 저장소·AI 파이프라인 사이의 비즈니스 흐름을 오케스트레이션한다. AI 추출·규칙·RAG를 중복 구현하지 않고 `ai/` 패키지를 호출한다.

## 현재 구조

최소 MVP의 계약·문서·추출·분석 흐름은 주로 `app/api/routes/`와 `app/workers/`에 구현돼 있다. `services/`에는 legacy minimum MVP 서비스가 있으며, 목표 계층으로의 전면 이관은 아직 하지 않았다.

이번 문서 정리는 계층 이동이나 동작 리팩터링을 수행하지 않는다.

## 유지할 경계

- 규칙 엔진 결과를 LLM·RAG가 변경하지 않는다.
- RAG 근거가 없으면 `evidence_sources=[]`를 유지한다.
- 규칙 결과와 생성 결과를 분리한다.
- 외부 AI 실패를 안전한 오류 또는 fallback으로 변환한다.
