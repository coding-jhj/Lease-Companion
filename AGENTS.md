# AGENTS.md — 슬기로운 계약생활 (lease-companion)

이 파일은 Codex·Claude Code 등 모든 AI 에이전트의 **단일 기준 지시서**다. 작업 전 반드시 읽는다.

## 프로젝트 목적

첫 전월세 계약을 준비하는 2030 청년 임차인이 계약서·특약 이미지를 업로드하면, AI가 핵심 정보를 추출하고 규칙 엔진이 확인 필요 항목을 분류하며 RAG가 공식 근거로 질문·체크리스트·행동을 제공한다.

- **서비스명**: 슬기로운 계약생활
- **대상 사용자**: 첫 전월세 계약을 준비하는 2030 청년 임차인
- **대상 계약**: 주거용 전세 / 보증부 월세 / 일반 월세
- **제외 범위**: 부동산 매매, 상가 임대차, 경매, 소송, 계약 종료·보증금 반환 분쟁, 거주 중 하자 대응

## 사용자 흐름 (기준 6단계)

1. 현재 계약 단계 선택 (계약서 확인 중 / 서명 직전 / 계약 직후)
2. 계약서·특약 이미지 업로드
3. AI 핵심 정보 추출 (임대인, 주소, 보증금, 계약 기간, 주요 특약)
4. 확인 필요 항목 분석 (누락, 모호, 문서 간 불일치, 추가 확인)
5. 질문과 서명 전 체크리스트 제공 (임대인·공인중개사 질문, 우선 확인, 체크리스트)
6. 계약 직후 행동과 결과 요약 (전입신고, 확정일자, 계약서·이체내역 보관, 권리 확보, 요약)

AI 추출 정보는 분석 전에 사용자가 **확인·수정**할 수 있어야 한다.

## PoC / MVP 범위 (혼용 금지)

- **PoC**: 계약서·등기사항증명서·건축물대장 입력 → 멀티모달 LLM 추출 → 필드 정규화 → Python 규칙 엔진 문서 간 비교 → RAG → 확인 항목·질문·체크리스트 생성
- **MVP**: 계약 단계 선택 → 이미지 업로드 → AI 추출 → 사용자 확인·수정 → 규칙 분류 → 질문·체크리스트 → 계약 직후 행동·요약

상세는 `docs/planning/poc-scope.md`, `docs/planning/mvp-scope.md` 참조. 문서와 코드에서 두 범위를 섞지 않는다.

## 최상위 폴더 책임

| 폴더 | 책임 |
|------|------|
| `ai/` | 멀티모달 추출, 정규화, 규칙 엔진, RAG, 생성, guardrail, AI 스키마·평가 |

| `data/` | 비식별 샘플, 스키마, 규칙, RAG 자료·메타데이터, 평가 데이터 |
| `docs/` | 기획·아키텍처·API·데이터·AI 설계 문서, 결정 기록, 회의록 |

## 컴포넌트 책임 구분

- **멀티모달 LLM**: 정보 **추출**만 담당한다.
- **Python 규칙 엔진**: 문서 비교·확인 필요 항목 **분류**를 담당한다. LLM이 규칙 결과를 임의로 변경하지 않는다.
- **RAG**: 공식 자료 기반 **근거 검색·설명**을 담당한다. 판정을 내리지 않는다.

## 원칙 (모든 에이전트 준수)

- AI는 계약 가능 여부, 안전 여부, 전세사기 여부, 법률적 결론을 **단정하지 않는다.** 확인 항목·질문·체크리스트·증빙 행동만 제공한다.
- 실제 개인정보·계약 문서를 Git에 커밋하지 않는다.
- 기존 파일과 사용자 변경사항을 임의로 삭제·덮어쓰지 않는다.
- 요청 범위를 벗어난 기능·기술을 임의로 추가하지 않는다.
- 미정 기술(프론트 프레임워크, DB, 벡터 DB, 배포, LLM 제공자·모델, 인증)은 임의 확정하지 않고 README/문서에 `TODO`로 표시한다.
- 작업 전 적용 범위의 `AGENTS.md`와 관련 문서를 확인한다.
- 변경 후 관련 테스트와 문서를 확인한다.
- 테스트하지 않은 내용을 테스트 완료라고 쓰지 않는다.
- `git commit` / `git push`는 사용자의 명시적 요청이 있을 때만 수행한다.
