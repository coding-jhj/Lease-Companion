# AI 파이프라인

## 단계 흐름

```
추출 → 정규화 → 규칙 → RAG → 생성 → Guardrail
```

| 단계 | 모듈 | 책임 |
|------|------|------|
| 추출 | `extraction` | 멀티모달 LLM으로 문서 핵심 필드 추출 |
| 정규화 | `normalization` | 주소·금액·날짜·이름을 비교 가능한 형태로 정규화 |
| 규칙 | `rules` | 문서 간 비교, 누락·모호·불일치·확인 필요 분류 |
| RAG | `rag` | 공식 법령·표준계약서·가이드에서 근거 검색 |
| 생성 | `generation` | 질문·체크리스트·계약 직후 행동·요약 생성 |
| Guardrail | `guardrails` | 단정 표현·잘못된 출력 제한 |

## 원칙

- 추출값(문서에서 읽은 값)과 생성값(모델이 만든 질문·요약)을 구분한다.
- LLM은 규칙 엔진 결과를 임의로 변경하지 않는다.
- 근거 부족 시 추측하지 않고 `정보 부족` / `확인 필요`로 반환한다.
- 모든 단계 출력은 구조화 스키마(`ai/src/lease_companion_ai/schemas/`)를 따른다.
- `pipelines`가 PoC·MVP 흐름을 각각 연결한다.
