# 시스템 아키텍처

## 계층 구성

```
[ frontend ]  사용자 화면 (스택 미정)
      │  HTTP (REST)
      ▼
[ backend ]   FastAPI — 세션·업로드·검증·상태·결과 API
      │  함수 호출 (ai/ 패키지)
      ▼
[ ai ]        추출 · 정규화 · 규칙 엔진 · RAG · 생성 · guardrail
      │  참조
      ▼
[ data ]      샘플 · 스키마 · 규칙 · RAG 자료 · 평가 데이터
```

## 책임 경계

- **frontend → backend**: REST 요청. 업로드·단계 선택·추출 확인·분석 요청·결과 조회.
- **backend → ai**: 백엔드는 오케스트레이션만. AI 로직을 중복 구현하지 않고 `ai/` 파이프라인을 호출한다.
- **ai → data**: 분석 기준 데이터(스키마·규칙·RAG 자료)·평가 데이터를 참조한다.

## 컴포넌트 역할

- 멀티모달 LLM: 추출
- 규칙 엔진: 문서 비교·확인 항목 분류
- RAG: 공식 근거 검색·설명
