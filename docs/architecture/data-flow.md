# 데이터 흐름

## 3구분 흐름

```
입력 데이터  →  처리 데이터  →  결과 데이터
```

## 입력 데이터

- 계약서, 특약, 등기사항증명서, 건축물대장, 기타 계약 관련 문서
- 실제 데이터는 Git 제외. 비식별 샘플만 `data/sample/`.

## 처리 데이터

- 추출값 (`data/processed/extracted-fields/`)
- 정규화 문서 (`data/processed/normalized-documents/`)
- RAG 청크 (`data/rag/chunks/`)

## 결과 데이터

- 확인 필요 항목
- 임대인·공인중개사 대상 질문
- 서명 전 체크리스트
- 계약 직후 권리 확보 행동
- 미완료 항목
- 결과 리포트

## 원칙

- 원본을 직접 덮어쓰지 않는다. 가공물은 별도 산출물로 남긴다.
- 실제 자료와 생성 데이터를 명확히 구분한다.
- 결과 리포트·중간 산출물은 운영 환경 생성물로 Git에서 제외한다.
