## 로컬 7B 서빙 고려사항 (선택적 실험 진행 시)

로컬 7B는 MVP 크리티컬 패스에서 제외되며, 상용 vs 로컬 성능비교를 위한 선택적 병렬 실험(파인튜닝 B/C안)을 진행하는 경우에만 아래를 고려한다.

- 우선 `ai/` 내부 인터페이스로 호출(동일 프로세스). 최상위 `model-service`·`inference-server` 임의 추가 금지.
- 추후 부하·자원 분리가 필요하면 `services/model-api`로 분리 가능(결정 대기).
- 서빙 런타임·GPU 요구·양자화 여부 미정 (TODO).

## 학습·실험 환경

- 파인튜닝은 Colab 등 외부 학습 환경 사용을 고려(방향만, 구체 환경 미정 — TODO).
- 학습 산출물(가중치·체크포인트)은 배포 아티팩트로 별도 관리하며 코드 저장소에 넣지 않는다.

## 상용 LLM API 의존

- 외부 API 의존은 네트워크·비용·지연·가용성 리스크를 가진다. routing fallback 정책과 연계 필요(모델 라우팅 문서 예정).
- 제공자·모델 확정: 조항 구조화 Gemini 3.5 Flash, 설명·질문·행동 생성 GPT-5.6 Sol. 리전 미정 (TODO).

## 비밀 관리

- API 키·DB 접속정보 등 비밀은 `.env`(또는 배포 플랫폼 시크릿)로 관리하고 저장소에 커밋하지 않는다.
- 비밀 관리 구체 방식(시크릿 매니저 등)은 배포 플랫폼 결정 후 확정 (TODO).

## 아티팩트 Git 제외 (확정)

- **모델 가중치·체크포인트는 Git에 커밋하지 않는다.** (루트 [`../../AGENTS.md`](../../AGENTS.md) 원칙)
