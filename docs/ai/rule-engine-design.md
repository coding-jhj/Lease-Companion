# 규칙 엔진 설계

## 역할

Python 규칙 엔진이 정규화된 추출값을 입력받아 문서 간 비교·확인 필요 항목 분류를 수행한다. LLM이 아닌 결정론적 규칙으로 동작한다.

## 입력

- 정규화된 추출 필드 (`ai/normalization` 출력)

## 결과 상태

- `누락` — 필수 필드가 없음
- `모호` — 조건·기한이 불명확
- `불일치` — 문서 간 값이 다름
- `확인 필요` — 추가 확인이 요구됨

## 원칙

- 규칙 엔진은 상태를 **분류**만 한다. 계약 안전·위험·사기·합법 여부를 단정하지 않는다.
- LLM은 규칙 엔진 결과를 임의로 변경하지 않는다.
- 각 규칙은 `rule_id·stage·input_fields·condition·result·source·version`을 갖는다. (`../data/rule-definition.md`)
