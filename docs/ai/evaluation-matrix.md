# 평가 지표 매트릭스

평가 지표를 표로 정의한다. **목표 수치는 없다.** 지표 정의·대상·측정 방법·기록 형식만 관리하고, 실제 측정값만 기록한다. 단계별 평가 구분은 [`evaluation-plan.md`](evaluation-plan.md) 참조.

## 기록 형식 (공통)

각 지표는 측정 시 다음을 함께 기록한다: `metric_id` · `측정 대상` · `데이터셋(split)` · `측정값` · `측정일` · `모델/설정 버전`. 측정 전에는 `측정값 = 미측정`으로 둔다.

## 지표 표

| # | 지표 | 대상 | 측정 방법 | 기록 형식 |
|---|------|------|-----------|-----------|
| 1 | 조항 유형 분류 P·R·F1 | `local_model` | 라벨 test set 대비 precision/recall/F1 (클래스별·매크로) | 클래스별·매크로 값, split 명시 |
| 2 | 명확성 분류 성능 | `local_model` | 명확/불명확/확인 필요 후보 분류 P·R·F1 또는 정확도 | 값 + split |
| 3 | 베이스 7B vs 파인튜닝 7B | `local_model` | 동일 test set에서 1·2 지표를 두 모델로 측정·대조 | 두 모델 값 나란히 |
| 4 | 구조화 JSON 스키마 준수율 | `local_model`·`extraction`·`generation` | 출력 중 스키마 검증 통과 비율 | 통과/전체, % |
| 5 | 로컬 모델 신뢰도 | `local_model` | 모델 신뢰도 분포·정오답 대비 보정(calibration) | 분포·구간별 정확도 |
| 6 | 로컬 처리율 | `routing` | 상용 LLM 재검토 없이 로컬에서 종결된 비율 | 로컬 종결/전체, % |
| 7 | 상용 LLM 재검토율 | `routing` | 저신뢰로 상용 LLM에 승격된 비율 | 승격/전체, % |
| 8 | 문서 간 일치·불일치 판정 정확도 | `rules` | 교차검증 판정을 정답 라벨 대비 정확도(9개 상태 기준) | 정확도 + 상태별 혼동표 |
| 9 | RAG 정답 근거 포함률 | `rag` | 정답 근거가 검색 결과 top-k에 포함된 비율 | 포함/전체, k 명시 |
| 10 | 인용 정확성 | `rag`·`generation` | 제시 근거가 실제 출처 내용과 일치하는 비율 | 일치/전체, % |
| 11 | 상용 LLM 단독 vs 전체 하이브리드 | `pipelines`·`routing` | 동일 end-to-end 입력에서 두 구성 결과 품질 대조 | 두 구성 값 나란히 |
| 12 | 문서당 처리 시간 | `pipelines` | end-to-end 처리 시간 (단계별 분해 가능) | 초, 단계별 선택 |
| 13 | 문서당 상용 API 비용 | `pipelines`·`providers` | 문서당 상용 LLM 호출 토큰·비용 | 비용/문서 |
| 14 | 근거 없는 출력·단정 표현 비율 | `generation`·`guardrails` | guardrail 위반(단정 표현·근거 없는 출력) 비율 | 위반/전체, % |

## 원칙

- 목표 수치를 임의로 생성하지 않는다. 정의·기록 형식만 확정한다.
- 평가 데이터는 train/validation/test를 분리해 `data/evaluation/` 하위에서 참조한다.
- 종합 판정(안전/위험/사기 점수)은 측정 대상이 아니다(사용하지 않음).
