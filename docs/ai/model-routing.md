# 모델 라우팅

`routing` 모듈이 단계별 처리 모델과 fallback을 선택하는 규칙을 정의한다.

## 원칙 (확정)

- **상용 LLM 우선.** 재검토·생성 단계는 상용 LLM을 우선 사용한다.
- **할당량 초과·장애 시 fallback.** 상용 LLM 호출 실패·할당량 소진·시간 초과 시 라우팅이 대체 경로(재시도, 대체 제공자, 저하 모드)를 결정한다.
- **로컬 7B는 상용 호출 전 1차 처리.** 조항 분류·명확성 후보는 로컬 7B가 먼저 처리한다.
- 규칙 엔진 판정은 라우팅 대상이 아니다. 결정론적으로 항상 수행한다.

## 단계별 처리 모델 선택

| 단계 | 처리 | fallback |
|------|------|----------|
| 문서 인식 (`ingestion`) | PDF 직접 추출 우선 → OCR | OCR 실패·저품질 시 VLM 보조 |
| 필드 추출 (`extraction`) | 상용 LLM/VLM | TODO (제공자 미정) |
| 조항 분류 (`local_model`) | 로컬 7B 1차 | 로컬 실패·저신뢰 시 상용 LLM 재검토로 승격 |
| 판정 (`rules`) | 규칙 엔진(고정) | 없음 (항상 수행) |
| 근거 (`rag`) | 벡터 검색 | 근거 없음 → `확인 불가`/`확인 필요` |
| 재검토·생성 (`providers`) | 상용 LLM 우선 | 할당량·장애 시 재시도·대체 제공자 |

## 로컬 7B ↔ 상용 LLM 경계

- **로컬 7B (`local_model`)**: 규칙 엔진 전 1차 분류. 최종 판정 안 함, 규칙 결과 변경 안 함.
- **상용 LLM (`providers`)**: 규칙 판정 후 저신뢰 결과 재검토, 근거 기반 쉬운 설명·질문·행동 생성. 규칙 판정 변경 안 함.
- 승격 트리거: 로컬 7B 저신뢰(임계값 미달) 또는 명확성 `확인 필요` 후보 → 상용 LLM 재검토.

## 관측 지표 (측정 후 기록)

로컬 처리율, 상용 LLM 재검토율, 문서당 상용 API 비용, 문서당 처리 시간. 정의·기록 형식: [`evaluation-matrix.md`](evaluation-matrix.md). 목표 수치는 측정 전 만들지 않는다.
