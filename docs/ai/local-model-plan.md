# 로컬 7B 모델 계획

파인튜닝한 로컬 7B 모델(`ai/src/lease_companion_ai/local_model/`)의 역할·인터페이스·경계를 정의한다.

## 역할 (확정)

상용 LLM 호출 전 **1차 분류** 단계. 계약서·특약 문장을 입력받아 조항 유형과 명확성 후보를 구조화 JSON으로 반환한다.

- 조항 유형 분류(`clause_type`)
- 명확성 분류(`clarity`: 명확 / 불명확 / 확인 필요 후보)
- 책임 주체(`responsible_party`)·조건(`condition`) 구조화
- 규칙 엔진·상용 LLM 처리 전 후보 신호 제공

## 하지 않는 것 (확정)

- 최종 법률 결론
- 규칙 엔진의 명시적 비교·판정 결과 변경
- 공식 근거 없는 사용자 행동 확정
- 종합 판정(안전/위험/사기 점수)

로컬 모델 출력은 **후보 신호**다. `classification`이 이 출력을 조항 유형·명확성 후보 구조로 정리해 후속에 넘긴다. 최종 판정은 규칙 엔진(`rules`), 근거는 RAG(`rag`), 저신뢰 재검토·생성은 상용 LLM(`providers`)이 담당한다.

## 입력 / 출력 (확정 형식)

- 입력: 계약서·특약 문장 (정규화 전 문장 단위 또는 조항 단위)
- 출력(구조화 JSON): `clause_type` · `clarity` · `responsible_party` · `condition` · `review_required`

출력 스키마는 `ai/src/lease_companion_ai/schemas/`와 파인튜닝 과제 정의(`fine-tuning-plan.md`)를 공유한다.

## 인터페이스 경계 (확정)

- `local_model/`은 추론 인터페이스만 노출한다. 문서 인식·추출·규칙·RAG·생성 로직을 포함하지 않는다.
- 파이프라인은 `routing`을 통해 `local_model`을 호출한다. 로컬 처리 실패·저신뢰 시 `routing`이 상용 LLM fallback을 결정한다. (`model-routing.md`)
- 결합도를 낮춰, 별도 모델 서버 배포가 확정되면 `services/model-api`로 분리할 수 있게 한다.

## 로컬 7B ↔ 상용 LLM 경계

| 구분 | 로컬 7B (`local_model`) | 상용 LLM (`providers`) |
|------|-------------------------|------------------------|
| 위치 | 규칙 엔진 전 1차 분류 | 규칙 판정 후 저신뢰 재검토·생성 |
| 산출 | 조항 유형·명확성 후보 | 쉬운 설명·질문·행동, 재검토 의견 |
| 판정 | 안 함 | 안 함 (규칙 판정 유지) |
| 우선순위 | 1차 처리 | 상용 우선 재검토·생성, fallback 포함 |

## 추후 분리 가능성

현재는 `ai/` 내부 `local_model/` 모듈. 모델 서버 배포·독립 스케일링이 필요해지면 `services/model-api`로 분리. 지금은 최상위 서비스를 추가하지 않는다.
