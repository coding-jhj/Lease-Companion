# AI 평가 계획

각 단계를 분리해 평가한다. 테스트는 `ai/tests/` 하위에서 단계별로 구분한다.

## 평가 구분

| 구분 | 대상 | 평가 관점 (초안) |
|------|------|------------------|
| 추출 | `extraction` | 핵심 필드 추출 정확도 |
| 규칙 | `rules` | 누락·모호·불일치·확인 필요 분류 정확도 |
| 검색 | `rag` | 공식 근거 검색 적합성 |
| 생성 | `generation` | 질문·체크리스트·행동·요약 품질, 단정 표현 여부 |
| 전체 흐름 | `pipelines` | end-to-end 결과 일관성 |

## 원칙

- 평가 데이터는 `data/evaluation/` 하위(extraction/rules/retrieval/generation/end-to-end)에 둔다.
- 근거 없는 성능 수치를 문서에 쓰지 않는다. 실제 측정값만 기록한다.
- guardrail 위반(단정 표현)은 생성 평가에서 반드시 확인한다.
