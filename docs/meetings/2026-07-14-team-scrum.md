## 2. 준비사항 점검 (10개 중 9개 완료)

### ① 계약 데이터셋 및 공개 자료 조사 — ✅
- 공식 근거·서식 15종 확보(주택임대차보호법·시행령, 표준 임대차계약서, 중개대상물 확인·설명서, 모의 등기사항증명서·건축물대장 서식, HUG 안내 등).
- **합성 데이터 40쌍 제작**: dev 30 + test 10 (계약서·등기사항증명서·건축물대장 번들, 전부 비식별).
- 결론: 데이터는 "수집"이 아니라 "합성 제작" 문제 → 재료로 직접 제작 완료.

### ② LLM·VLM 후보 모델 목록과 선정 근거 — ✅
최종 모델·기술 선정표(13단계) 확정:
- OCR: PaddleOCR-VL-1.6 (VLM 통합) / 디지털 PDF: PyMuPDF·PDF.js
- 조항 구조화·필드 추출: Gemini 3.5 Flash
- 설명·질문·행동 생성: GPT-5.6 Sol
- 근거 검색: gemini-embedding-001 + BM25 → 재정렬 Cohere rerank-v4.0-pro
- 로컬 7B: 크리티컬 패스 제외 → 선택적 성능비교 실험

### ③ OCR 및 PDF 처리 방식 — ✅
텍스트 레이어 확인 → 있으면 직접 추출(PyMuPDF·PDF.js), 없으면 OCR(PaddleOCR-VL) → 구조화. 표·레이아웃은 PaddleOCR-VL의 VLM 통합으로 처리(별도 VLM 단계 없음).

### ④ RAG 데이터 구성안 — ✅
- 공식 근거 15종 코퍼스 + 규칙별 근거 매핑(rule_evidence_map).
- 파이프라인: 임베딩(gemini-embedding-001) + BM25 하이브리드 → Cohere 리랭크.
- 검증: 규칙별 기대 근거 goldset 확보. (URL·시행일 일부 확인 필요)

### ⑧ 프로젝트의 기술적 차별성 한 문장 — ✅
> 범용 LLM은 못 하는 **계약서·등기사항증명서·건축물대장 3문서 교차검증**과 **등기 권리탐지 규칙 엔진**(근저당·압류·신탁)에 공식 법령 RAG 근거를 붙여, ‘안전/위험 단정’ 대신 **근거 있는 확인 질문·행동**을 제공하는 첫 전월세 임차인용 계약 리스크 도우미.

핵심 레버: 문서 교차검증 · 등기 권리탐지 규칙 엔진 · 개인정보 비식별 파이프라인.

### ⑨ 상용 LLM과 로컬 모델의 운영 구조 — ✅
- 무거운 추론은 상용 API, 개인정보·최종 판정은 전부 로컬.
- 개인정보 비식별 게이트: API로 보내기 전 로컬 토큰화 → 비식별 텍스트만 전송 → 로컬 복원 → 세션 종료 시 삭제.
- Fallback: 할당량·장애 시 Claude CLI·Codex CLI로 분기.
- 로컬 7B는 성능비교 실험(선택). 로컬 실행은 PaddleOCR-VL(0.9B)뿐 → GPU 부담 작음.

### ⑩ GPU 제공 여부와 학습 환경 — ❌ 미결
팀 확인 필요. 로컬 7B 크리티컬 패스 제외로 무거운 GPU는 불필요, 성능비교 실험 진행 여부와 연결.
