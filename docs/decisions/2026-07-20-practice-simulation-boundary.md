# 계약 연습 시뮬레이션의 제품·데이터 경계

- 날짜: 2026-07-20
- 상태: 확정

> 구현 갱신(2026-07-22): 이 ADR의 데이터·판정 경계를 유지한 채 승인 시나리오 3개가 AI 평가, Backend 영속 API, Frontend 텍스트 대화·복기, 실제 API Playwright 흐름으로 연결됐다. 아래의 “구현 전”과 첫 시나리오 1개 표기는 결정 당시의 역사적 상태다. 미디어·음성은 여전히 후속 범위다.

## 배경

슬기로운 계약생활의 기존 MVP는 로그인 사용자가 실제 계약 건과 문서를 8단계로 점검하는 흐름이다. 새 계약 연습 시뮬레이션은 같은 계약 규칙·공식 근거를 활용하지만, 합성 상황에서 가상 임대인·공인중개사와 대화하고 사용자의 확인 행동을 복기하는 별도 목적을 가진다.

연습을 기존 계약 분석 흐름에 그대로 넣으면 다음 문제가 생긴다.

- 합성 시나리오가 실제 `contract_id`·문서·분석 이력과 섞인다.
- 계약 내용 판정과 사용자 대화 행동 평가의 책임이 혼합된다.
- 런타임 LLM이 무제한으로 상황과 정답을 생성하면 결과 재현성과 평가 기준이 흔들린다.
- 영상 압박 연출을 실제 기한이나 계약 추천으로 오해할 수 있다.

## 검토한 선택지

### 선택지 A. 런타임 LLM이 계약 상황과 대화를 모두 생성

- 장점: 상황 다양성이 크다.
- 단점: 정답·규칙·근거의 재현성이 낮고 비용·지연·안전 검수가 어렵다.

### 선택지 B. 승인된 합성 시나리오 카탈로그에서 선택

- 장점: 규칙 결과·대화·영상·정답표를 버전별로 검증할 수 있다.
- 단점: 초기 시나리오 제작 비용이 들고 다양성이 카탈로그 크기에 제한된다.

### 선택지 C. 기존 실제 계약 분석 실행을 연습 세션으로 재사용

- 장점: 새 데이터 모델이 적다.
- 단점: 실제 계약과 합성 연습의 식별자·저장·평가 책임이 섞이고 사용자에게 혼란을 준다.

## 결정

### 1. 별도 사용자 모드

- `실전 계약 점검`의 기준 8단계는 유지한다.
- `계약 연습`은 대시보드에서 분리해 진입하는 병렬 확장 모드로 둔다.
- 계약 연습은 아직 구현 전이며, 첫 세로 흐름 검증 후 범위를 확장한다.

### 2. 승인된 합성 시나리오

- 선택지 B를 채택한다.
- 첫 구현은 `PRACTICE-BROKER-PRESSURE-001` 한 개로 고정한다.
- 이후 무작위 선택은 승인·버전 관리된 시나리오 카탈로그 안에서만 수행한다.
- 런타임 LLM은 계약 사실·권리관계·정답을 임의 생성하지 않는다.

### 3. 식별자와 저장 분리

- 실제 사용자 계약에는 기존 `contract_id`를 사용한다.
- 합성 평가 사례에는 기존 `case_id`를 사용한다.
- 연습에는 `scenario_id`, `scenario_version`, `practice_session_id`, `practice_turn_id`를 사용한다.
- 연습 세션은 실제 계약 분석 실행·체크리스트·피드백 이력과 분리해 저장한다.
- 필요하면 시나리오 내부의 합성 계약 입력은 canonical 계약 스키마를 재사용하되 실제 `contract_id`로 승격하지 않는다.

### 4. 판정 권한 분리

- Python 계약 규칙 엔진만 합성 계약 내용의 `RuleStatus`와 `urgency`를 결정한다.
- 연습 행동 평가기는 사용자가 필요한 질문·보류·확인 행동을 수행했는지 별도 결과로 판단한다.
- Gemini 답변 분류·생성과 RAG는 계약 규칙 결과를 변경하지 않는다.
- provider 실패·timeout·형식 오류는 사용자 오답으로 처리하지 않고 `확인 필요` fallback으로 처리한다.

### 5. 영상 제작과 재생

- 캐릭터 기준 이미지는 Gemini로 생성한다.
- 영상은 실시간 아바타가 아니라 상태별 사전 제작 클립으로 제공한다.
- 상황 설명·기본 대기·압박 대기·재촉·긍정·회피·전환 클립을 manifest로 관리한다.
- image-to-video와 음성합성 제공자는 현재 미정이며 별도 결정 전 설치·확정하지 않는다.

### 6. 결과와 안전 표현

- 숫자형 안전·위험·사기 점수와 종합 계약 결론을 제공하지 않는다.
- 검증된 계산 규칙이 없는 예상 금전 피해액을 표시하지 않는다.
- 결과는 잘한 행동·놓친 확인·추천 문장·다음 행동·공식 근거로 제공한다.
- `가상 연습` 표시, 연습 종료·건너뛰기·음소거·자막·움직임 줄이기를 제공한다.

### 7. 첫 MVP 입력 범위

- 첫 버전은 텍스트 답변과 행동 선택을 사용한다.
- 실시간 음성 입력·STT·카메라 분석은 후속 범위다.
- 사용자가 입력을 시작하면 압박 강도 상승을 멈춘다.

## 근거

- 합성 시나리오와 정답표를 고정해야 같은 입력에 같은 평가가 가능하다.
- 계약 규칙과 사용자 행동 평가를 분리해야 기존 판정 권한 원칙을 유지할 수 있다.
- 사전 제작 클립은 PC·모바일 품질과 접근성을 검수하기 쉽고 런타임 비용을 제한한다.
- 실제 계약 이력과 연습 이력을 분리해야 사용자 데이터 의미와 접근 제어가 명확하다.
- 행동 기반 복기는 종합 안전 결론 없이도 실제 계약 대응 능력을 연습하게 한다.
