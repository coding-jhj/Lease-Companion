# 개인정보 처리 원칙 (개발 단계)

> 본 문서는 개발 단계의 **데이터 취급 원칙**이다. 서비스 운영용 개인정보 처리방침은 별도 확정 필요(아래 TODO).

## 커밋 금지 대상

- 실제 계약서·등기사항증명서·건축물대장·중개대상물 확인설명서 등 개인정보 포함 원본을 Git에 커밋하지 않는다.
- 이름·주소·연락처·주민등록번호·계좌번호 등 민감정보를 커밋하지 않는다.
- **모델 가중치·체크포인트·어댑터 파일**을 커밋하지 않는다. 메타데이터만 `data/model-metadata/`에 둔다.
- 생성된 대용량 벡터 인덱스를 커밋하지 않는다.

## Git 제외 경로 (`.gitignore` 반영됨)

- `data/raw/*` — 실제 원본 전체 제외 (`data/raw/README.md`만 예외)
- `data/uploads/`, `uploads/` — 업로드 원본
- `data/processed/**/*` — 가공 산출물 (각 폴더 `README.md`만 예외)
- `data/rag/index/`, `*.faiss`, `*.index`, `*.pkl`, `chroma/`, `*.sqlite3` — 벡터 인덱스
- `*.safetensors`, `*.gguf`, `*.bin`, `*.pt`, `*.ckpt`, `adapter_model.*`, `**/checkpoints/` — 모델 가중치

## 샘플·데이터셋

- `data/sample/`, `data/datasets/`에는 **가상 또는 완전히 비식별화한 데이터만** 둔다.
- 파인튜닝 데이터셋도 비식별·합성 원칙을 따른다([`training-dataset.md`](training-dataset.md)).
- 실제 자료와 생성 데이터를 명확히 구분한다. 실제 원본에서 파생한 비식별본은 재식별 불가함을 확인한 뒤 둔다.

## 로그·오류

- 개인정보·문서 내용을 로그·오류 메시지에 남기지 않는다(backend 규칙).
- 프롬프트·중간 산출물을 저장할 때 개인정보를 제거하거나 마스킹한다.
