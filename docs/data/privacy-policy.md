# 개인정보 처리 원칙

## 실제 계약 문서·개인정보

- 실제 계약서·등기사항증명서·건축물대장 등 개인정보 포함 원본을 Git에 커밋하지 않는다.
- 이름·주소·연락처·계좌번호 등 민감정보를 커밋하지 않는다.
- `data/raw` 내부 실제 데이터는 Git에서 제외한다. (`.gitignore`)

## 샘플 데이터

- `data/sample/`에는 가상 또는 완전히 비식별화한 데이터만 둔다.
- 실제 자료와 생성 데이터를 명확히 구분한다.

## 로그·오류

- 개인정보·문서 내용을 로그·오류 메시지에 남기지 않는다. (backend 규칙)
