# 규칙 정의

문서 내부 판정과 문서 교차검증에 쓰는 규칙 데이터의 필드와 예시. 규칙 엔진은 [`judgment-spec.md`](judgment-spec.md)의 J01–J12를 **명시적 최종 판정**으로 구현한다. LLM·로컬 모델은 규칙 결과를 임의로 바꾸지 않는다.

## 규칙 레코드 필드

| 필드 | 의미 |
|------|------|
| `rule_id` | 규칙 식별자 |
| `judgment_id` | 연결 판정 (J01–J12) |
| `stage` | 적용 계약 단계 (user-flow 3단계 "계약 단계 선택" 입력값. **값 목록 미확정 — TODO**) |
| `input_fields` | 규칙이 사용하는 입력 필드 (`document-fields.md` 기준) |
| `condition` | 판정 조건 |
| `result` | 결과 상태 (공통 9개 중, 판정별 허용 집합) |
| `urgency` | 시급도 (5개 중 1개) |
| `source` | 근거 유형 (공식 자료) |
| `version` | 규칙 버전 |

## 결과 상태 (공통 9개)

`일치` · `불일치` · `명확` · `불명확` · `미기재` · `상충 가능` · `확인 필요` · `확인 불가` · `적용 제외`

- 규칙 엔진은 상태를 **분류만** 한다. 안전·위험·사기 여부를 단정하지 않는다.
- 판정마다 허용 상태 집합이 다르다. 세부는 [`judgment-spec.md`](judgment-spec.md) 표를 단일 기준으로 한다.

## 핵심 규칙 예시 (초안)

> 예시는 구조 설명용이다. 실제 규칙·근거·조건은 공식 자료 확인 후 확정한다.

- **미기재(J06)**: 계약 유형별 필수 금액 필드(보증금·계약금·잔금 등)가 추출되지 않으면 `미기재`.
- **불일치(J01)**: 계약서 임대인과 등기 소유자 이름이 정규화 후 다르면 `불일치`. 등기 없음이면 `확인 불가`.
- **불명확(J10)**: 보증금 반환 시점·조건 특약이 조건·기한을 특정하지 않으면 `불명확`. 관련 조항 자체가 없으면 `미기재`.
- **상충 가능(J12)**: 동일 사안에 대해 본문과 특약이 어긋나면 `상충 가능`. 특약이 없으면 `적용 제외`.
