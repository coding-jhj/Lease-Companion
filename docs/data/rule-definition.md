# 규칙 정의

문서 비교·확인 필요 항목 분류에 쓰는 규칙 데이터의 필드와 예시.

## 규칙 레코드 필드

| 필드 | 의미 |
|------|------|
| `rule_id` | 규칙 식별자 |
| `stage` | 적용 단계 (계약서 확인 중 / 서명 직전 / 계약 직후) |
| `input_fields` | 규칙이 사용하는 입력 필드 |
| `condition` | 판정 조건 |
| `result` | 결과 상태 (누락 / 모호 / 불일치 / 확인 필요) |
| `source` | 근거 (공식 자료) |
| `version` | 규칙 버전 |

## 핵심 규칙 예시 (초안)

> 예시는 구조 설명용이다. 실제 규칙·근거는 공식 자료 확인 후 확정한다.

- **누락**: 필수 필드(보증금·계약 기간 등)가 추출되지 않으면 `확인 필요`로 분류.
- **불일치**: 계약서 임대인과 등기사항증명서 소유자가 다르면 `확인 필요`로 분류.
- **모호**: 특약 문구가 조건·기한을 명시하지 않으면 `모호`로 분류.

## 결과 상태

- `누락` / `모호` / `불일치` / `확인 필요`
- 규칙 엔진은 상태를 분류만 한다. 안전·위험을 단정하지 않는다.
