# 판정 명세 (Judgment Spec) — 4영역 12항목

> 슬기로운 계약생활의 **판정 단일 기준**. 규칙 엔진([`rule-definition.md`](rule-definition.md))이 이 명세의 판정을 구현하며, 입력 필드는 [`document-fields.md`](document-fields.md)를 따른다.
> AI는 계약 가능·안전·전세사기·적법 여부를 **단정하지 않는다.** 확인 항목·질문·체크리스트·행동만 제공한다. 종합 점수(안전/위험/사기 가능성)는 만들지 않는다.

## 판정 구성요소

각 판정(J01–J12)은 다음을 갖는다.

1. **입력**: 필요한 문서·필드 (없으면 `확인 불가` 또는 `확인 필요`)
2. **판정 로직 요약**: 비교·검사 방식 (명시적 최종 판정은 규칙 엔진 담당)
3. **적용 가능한 결과 상태**: 아래 9개 공통 상태 중 해당 판정에 유효한 부분집합만
4. **기본 시급도**: 5개 중 1개
5. **데이터 부족 시 처리**: 입력이 없거나 불완전할 때의 반환
6. **공식 근거 연결**: RAG로 붙일 근거 유형(판정 자체는 근거로 내리지 않고, 설명·행동에 근거를 붙인다)

## 공통 결과 상태 (9개)

`일치` · `불일치` · `명확` · `불명확` · `미기재` · `상충 가능` · `확인 필요` · `확인 불가` · `적용 제외`

- **12개 판정을 동일 상태로 표현하지 않는다.** 판정마다 적용 가능한 상태 집합이 다르다(아래 표).
- 비교형 판정(A영역, J07·J08)은 `일치`/`불일치`를 축으로 한다.
- 명확성 판정(C영역, J09·J10·J11)은 `명확`/`불명확`을 축으로 한다.
- `적용 제외`는 **구조적으로 해당 없음**일 때만 쓴다(예: 전세에 월세 필드, 본인 계약, 특약 없음). "확인했고 문제없음"에는 쓰지 않는다 — 이 경우는 `일치`/`명확`으로 표기해 깨끗한 계약이 "확인됨"으로 읽히게 한다.
- `확인 불가`(입력 문서 자체가 없음)와 `확인 필요`(문서는 있으나 값이 불완전·모호·사용자 대면 확인 대상)를 구분한다.

## 시급도 (5개, 판정 상태와 별도)

`즉시 확인` · `계약 전 확인` · `계약 직후 조치` · `참고` · `분석 불가`

- 시급도는 판정이 참일 때 **사용자가 언제 움직여야 하는가**를 나타낸다(D영역 행동 안내와 연결).
- 입력 부족으로 `확인 불가`/`분석 불가`가 나오면 시급도는 `분석 불가`.

## A. 계약 당사자·목적물

| 판정 | 입력 문서·필드 | 판정 로직 요약 | 적용 가능 상태 | 기본 시급도 | 데이터 부족 시 |
|------|----------------|----------------|----------------|-------------|----------------|
| **J01** 계약서 임대인 = 등기 소유자 | 계약서 임대인 성명, 등기 소유자 성명(정규화 이름) | 두 이름 정규화 후 일치 비교. 대리인 계약이면 J04로 위임 | `일치` / `불일치` / `확인 필요` / `확인 불가` | 즉시 확인 | 등기 없음 → `확인 불가`; 대리인 계약 → `확인 필요` |
| **J02** 목적물 주소 일치 | 계약서 주소, 등기 소재지 주소(정규화 주소) | 주소 정규화 후 일치 비교. 확인설명서·건축물대장은 참고 보강 | `일치` / `불일치` / `확인 필요` / `확인 불가` | 계약 전 확인 | 등기 없음 → `확인 불가`; 표기 불명확 → `확인 필요` |
| **J03** 공동소유 여부·추가확인 | 등기 공동소유자 여부·지분 | 등기상 공동소유면 전원 동의·서명 확인 필요로 표시 | `확인 필요` / `적용 제외` / `확인 불가` | 계약 전 확인 | 등기 없음 → `확인 불가`; 단독소유 → `적용 제외` |
| **J04** 대리인 계약·권한 서류 확인 | 계약서 대리인 성명·관계, 권한 서류 언급 | 대리인 계약이면 위임장·인감증명 등 권한 서류 확인 필요로 표시 | `확인 필요` / `적용 제외` / `확인 불가` | 즉시 확인 | 대리 여부 불명확 → `확인 필요`; 본인 계약 → `적용 제외` |
| **J05** 입금 계좌 명의 = 계약 상대 | 계약서 입금 계좌 명의, 임대인/소유자 성명 | 계좌 명의와 계약 상대(임대인=소유자) 이름 일치 비교 | `일치` / `불일치` / `확인 필요` / `확인 불가` | 즉시 확인 | 계좌 정보 미기재 → `확인 필요`; 상대 이름 확인 불가 → `확인 불가` |

## B. 금액·기간

| 판정 | 입력 문서·필드 | 판정 로직 요약 | 적용 가능 상태 | 기본 시급도 | 데이터 부족 시 |
|------|----------------|----------------|----------------|-------------|----------------|
| **J06** 보증금·월세·계약금·잔금 기재 상태 | 계약서 보증금·차임·계약금·잔금 | 계약 유형별 필수 금액 필드가 모두 기재됐는지 검사(전세는 월세 필드 제외) | `명확` / `미기재` / `확인 필요` / `적용 제외` | 계약 전 확인 | 필수 금액 누락 → `미기재`; 유형상 없는 필드 → `적용 제외` |
| **J07** 금액 숫자·한글 표기 일치 | 보증금·차임의 숫자 표기, 한글 표기 | 숫자값과 한글 표기 금액을 파싱해 일치 비교 | `일치` / `불일치` / `확인 필요` / `확인 불가` | 계약 전 확인 | 한글 표기 없음 → `확인 필요`; 파싱 불가 → `확인 불가` |
| **J08** 지급일·입주일·계약 기간 일관성 | 계약금/잔금 지급일, 입주일, 계약 시작·종료일 | 날짜 순서·계약 기간 정합성 검사(잔금·입주일·시작일 모순 여부) | `일치` / `불일치` / `미기재` / `확인 필요` | 계약 전 확인 | 날짜 누락 → `미기재`; 해석 모호 → `확인 필요` |
| **J09** 관리비 금액·포함 항목 명확성 | 관리비 금액, 포함 항목 | 금액·포함 항목이 특정되는지 검사. 관리비 항목 자체가 없으면 제외 | `명확` / `불명확` / `미기재` / `적용 제외` / `확인 필요` | 참고 | 항목 없음 → `적용 제외`; 금액만·항목 모호 → `불명확` |

|------|----------------|----------------|----------------|-------------|----------------|
| **J10** 보증금 반환 시점·조건 명확성 | 보증금 반환 관련 특약·본문 | 반환 시점·조건이 특정되는지 검사(모호어·조건 누락 탐지) | `명확` / `불명확` / `미기재` / `확인 필요` | 계약 전 확인 | 관련 조항 없음 → `미기재`; 조건 모호 → `불명확` |
| **J11** 수리·원상복구 책임 명확성 | 수리·원상복구 책임 특약·본문 | 책임 주체(임대인/임차인)·범위가 특정되는지 검사 | `명확` / `불명확` / `미기재` / `확인 필요` | 참고 | 관련 조항 없음 → `미기재`; 주체·범위 모호 → `불명확` |
| **J12** 본문–특약 상충 가능성 | 계약서 본문 조항, 특약 조항 전체 | 동일 사안(기간·금액·책임 등)에 대해 본문과 특약이 어긋나는지 검사 | `상충 가능` / `명확` / `확인 필요` / `적용 제외` | 즉시 확인 | 특약 없음 → `적용 제외`; 상충 판단 애매 → `확인 필요` |

## D. 행동 안내 (판정 상태가 아니라 산출물)

D영역은 J01–J12 결과를 **사용자 행동으로 변환**한다. 자체 결과 상태를 갖지 않으며, 시급도와 계약 단계에 따라 묶는다.

| 산출물 | 트리거 | 시급도 연결 |
|--------|--------|-------------|
| 단계별 미확인 항목 | `확인 필요`·`확인 불가`·`불명확`·`미기재` 판정 모음 | 판정별 시급도 승계 |
| 계약금 입금 전 질문 | J01·J04·J05·J07 등 금전·당사자 위험 | `즉시 확인` |
| 서명 전 체크리스트 | J02·J06·J08·J09·J10·J11·J12 | `계약 전 확인` |
| 계약 직후 권리 확보 행동 | 전입신고·확정일자 등(판정과 별개 고정 안내) | `계약 직후 조치` |
| 보관 대상 안내 | 계약서·이체내역·대화 기록 | `참고` |

## 공식 근거 연결 방식

- 판정 자체는 규칙 엔진이 문서 비교로 내린다. **RAG 근거로 판정을 바꾸지 않는다.**
- 각 판정에는 근거 유형 태그를 붙여, 설명·질문·행동 생성 시 RAG가 해당 유형의 공식 자료를 검색해 근거를 첨부한다.
- 근거가 없으면 판정을 유지하되 설명을 `근거 확인 필요`로 표시한다.

| 판정군 | 근거 유형(예) |
|--------|---------------|
| J01–J05 | 등기·소유·대리 관련 안내(공공기관 가이드) |
| J06–J09 | 표준 주택임대차계약서 기재 항목, 관리비 관련 안내 |
| J10–J12 | 주택임대차보호법·표준계약서 특약·원상복구 관련 조항 |
| D 계약 직후 | 전입신고·확정일자·대항력·우선변제 관련 법령·가이드 |

근거 출처 기준은 [`rag-sources.md`](rag-sources.md), 메타데이터는 [`../../data/rag/metadata/`](../../data/rag/metadata/).
