# 발표 MVP와 해커톤 고도화 범위

- 작성일: 2026-07-23
- 목적: 7월 31일 발표 범위와 8월 10일 해커톤 이후 확장 계획을 섞지 않고 설명한다.

## 7월 31일 발표 MVP

발표에서는 현재 코드와 검증 결과로 재현할 수 있는 기능만 시연한다.

1. 회원가입·로그인과 계약 건 저장
2. 계약 상황 입력과 계약서·관련 문서 업로드
3. 문서에서 읽은 값의 사용자 확인·수정
4. Python 규칙 엔진의 항목별 판정과 공식 근거 연결
5. 확인 우선순위·질문·수정 요청·단계별 행동 제공
6. 승인된 합성 시나리오 3개의 계약 대화 미션
7. 사용자 답변·최종 행동 저장과 행동 기반 복기
8. 체크리스트와 계약 직후 행동 저장·재조회

발표에서는 계약의 안전성·사기 여부·적법성을 확정하지 않는다. 실제 외부 provider 품질·비용을 검증하지 않은 항목은 완료로 표현하지 않는다.

## 해커톤 고도화 계획

다음 항목은 현재 발표 MVP의 완료 조건에 포함하지 않는다.

1. 피해 유형별 승인 시나리오 카탈로그 확대
2. 시나리오별 발화·재촉·반응 사전 제작 영상 교체
3. 계약금·가계약금 송금 전 빠른 점검 흐름
4. Gemini·Cohere 실제 호출 품질·비용·429 부하 검증
5. 운영 배포, 비밀정보·토큰·문서 보관 정책 확정
6. 외부 공공 데이터 연동과 현재 `확인 불가` 항목 자동화
7. 접근성 기능(음소거·자막·움직임 줄이기)과 미디어 품질 검수

## 발표 시 구현 상태 표현

- `구현 완료`: 로컬 통합 흐름과 회귀 테스트로 확인한 기능
- `부분 구현`: 인터페이스와 fallback은 있으나 실제 provider·미디어 품질을 검증하지 않은 기능
- `후속 계획`: 코드·운영 환경·근거 검증 중 하나라도 완료되지 않은 기능

사전 제작 영상 manifest는 현재 공통 상태 영상에 연결되어 있다. 시나리오별 고유 영상 제작은 해커톤 고도화 범위다.
