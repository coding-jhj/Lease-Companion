# 최소 MVP 실행 안내

## 현재 구현 범위

브라우저에서 계약서와 등기사항증명서를 업로드하고, 추출값을 확인·수정한 뒤 R01~R10 결과와 질문·행동·근거 후보를 확인한다.

- 입력: 텍스트 레이어가 있는 PDF 2개. 개발 확인용 UTF-8 TXT도 허용한다.
- 추출: 표준 양식의 표제와 핵심 문구를 읽는 결정론적 파서.
- 판정: `data/rules/rule_spec.csv`를 사용하는 Python 규칙 엔진.
- 근거: `data/rules/source_inventory.csv`의 메타데이터. 확인된 URL만 링크한다.
- 저장: 단일 브라우저 세션만 사용하며 문서와 결과를 영속 저장하지 않는다.

스캔·사진 PDF OCR, 상용 LLM, 회원·계약 건·DB 저장은 이 최소 MVP에 포함하지 않는다.

## 설치와 실행

PowerShell에서 저장소 루트를 기준으로 실행한다.

```powershell
python -m pip install -r requirements-minimum-mvp.txt
.\scripts\run-minimum-mvp.ps1
```

브라우저에서 `http://127.0.0.1:8000`을 연다. 샘플 검증은 `data/sample/contracts/contract_001.txt`와 `data/sample/registry-records/registry_001.txt`를 사용할 수 있다.

## 사용자 흐름

1. 계약서와 등기사항증명서를 선택한다.
2. `정보 추출하기`를 누른다.
3. 문서 원문과 추출값을 대조하고 필요한 값을 수정한다.
4. 확인 완료에 체크한 뒤 `규칙 분석하기`를 누른다.
5. 항목별 상태·시급도·질문·행동·근거 후보를 확인한다.

## 제한사항

- 정규식 파서는 지원하는 표준 표제와 표현에서만 동작한다. 인식하지 못한 값은 사용자가 수정해야 한다.
- 근저당권·압류·가압류·신탁의 존재는 사실 플래그이며 안전·위험·적법 여부를 확정하지 않는다.
- 등기 발급일은 추출만 한다. 최신성 일수 기준은 정하지 않는다.
- 개인정보가 포함된 실제 문서를 저장소에 추가하거나 커밋하지 않는다.
