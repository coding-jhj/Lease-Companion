# 계약 연습 시뮬레이션 제품 규칙

- 날짜: 2026-07-20
- 상태: 0단계 기획 확정, 구현 전
- 서비스: 슬기로운 계약생활
- 첫 시나리오: `PRACTICE-BROKER-PRESSURE-001`

## 1. 해결할 문제

첫 전월세 계약자는 확인해야 할 내용을 알고 있어도 임대인·공인중개사가 결정을 재촉하는 실제 대화에서는 질문을 빠뜨리거나 확인 전 송금을 선택할 수 있다. 계약 연습 시뮬레이션은 합성 계약 상황과 가상 상대방을 이용해 이러한 대화를 안전하게 연습하고, 사용자가 수행한 확인 행동과 놓친 행동을 공식 근거와 함께 복기하도록 한다.

이 기능은 법률 상담, 계약 체결 추천, 실제 상대방과의 통화, 계약 안전·사기 여부 판정을 제공하지 않는다.

## 2. 대상 사용자와 사용 시점

- 대상: 첫 전월세 계약을 준비하는 2030 청년 임차인
- 사용 시점: 실제 계약서를 올리기 전 또는 실전 계약 점검과 별도로 대화 대응을 연습할 때
- 인증: 로그인 사용자만 연습 세션을 시작하고 자신의 결과를 다시 본다.

## 3. 첫 완료 목표

로그인 사용자가 `공인중개사의 즉시 계약 압박` 시나리오 한 개를 PC와 모바일에서 시작하고, 질문에 답하고, 행동을 선택하고, 복기 결과를 확인한 뒤 다시 열람할 수 있어야 한다.

첫 완료 목표에는 다음이 포함된다.

1. `실전 계약 점검`과 `계약 연습`의 분리된 진입점
2. 합성 계약 상황과 `가상 연습` 표시
3. 가상 공인중개사 영상·대사와 사용자 텍스트 답변
4. 미응답·애매함·적절한 확인에 따른 반응 변화
5. 진행·추가 확인·보류·중단 중 사용자 행동 선택
6. 잘한 행동·놓친 확인·추천 문장·다음 행동·공식 근거 복기
7. 연습 결과 저장·재조회

## 4. 첫 시나리오 정의

### 상황

가상 공인중개사가 다른 방문 예정자가 있다는 이유로 사용자에게 지금 계약 결정을 요구한다. 화면에는 합성 계약 상황과 가상 인물임을 명확히 표시한다.

대표 압박 대사:

> 다른 사람도 보러 오기로 해서요. 지금 결정하셔야 합니다.

이 문장은 연습용 대본이며 실제 제한시간이나 계약 마감이 아니다.

### 사용자 학습 목표

사용자는 압박에 즉시 동의하는 대신 다음 확인 행동을 수행하는 연습을 한다.

| 행동 ID | 기대 행동 | 인정 기준 |
|---|---|---|
| `PA01` | 즉시 결정을 유예한다 | 확인 후 결정하겠다는 의사를 분명히 말함 |
| `PA02` | 등기사항증명서와 소유자를 확인한다 | 최신 등기사항증명서와 계약 상대방·소유자 일치 확인을 요청함 |
| `PA03` | 특약과 보증금 반환 조건을 문서로 확인한다 | 구두 설명이 아니라 계약서·특약 문구 확인을 요청함 |
| `PA04` | 확인 전 계약금·가계약금 송금을 보류한다 | 필요한 확인이 끝나기 전 송금하지 않겠다고 말하거나 행동으로 선택함 |

한 답변에 모든 행동을 요구하지 않는다. 대화 여러 턴과 행동 선택을 합쳐 수행 여부를 판단한다.

### 답변 상태

| 상태 | 의미 | 시스템 반응 |
|---|---|---|
| `적절한 확인` | 현재 질문과 관련된 확인·보류 행동을 명확히 표현 | 긍정 반응 후 다음 질문 |
| `부분 확인` | 필요한 의미 일부만 표현 | 인정한 부분을 유지하고 빠진 내용을 다시 질문 |
| `애매한 답변` | 생각해 보겠다는 표현만 있고 확인 행동이 없음 | 회피·재압박 대사 후 선택지를 다시 제시 |
| `회피` | 질문과 무관하거나 답변을 피함 | 짧은 재질문 후 다음 시도 기회 제공 |
| `미응답` | 정해진 연습 시간 동안 답변 없음 | 대기 강도 상승 후 재촉 대사 재생 |
| `확인 필요` | provider 실패·형식 오류 등으로 의미 판별 불가 | 오답 처리하지 않고 다시 답할 기회 제공 |

## 5. 대기·압박 연출 규칙

1. 0~5초: 조용한 기본 대기 루프
2. 5~10초: 시계나 계약서를 확인하는 약한 압박 루프
3. 10초 이후: 재촉 대사 1회 후 대기 상태로 복귀
4. 사용자가 입력을 시작하면 압박 강도 상승을 멈추고 경청 상태로 전환
5. 사용자가 답변을 제출하면 대기 루프를 즉시 종료
6. 같은 재촉 대사를 무한 반복하지 않고 반복 횟수에 상한을 둠

표시되는 시간은 학습 연출이다. 실제 계약 기한으로 오해할 수 있는 문구를 사용하지 않는다.

## 6. 사용자 행동과 종료 조건

첫 시나리오에서 제공할 행동은 다음 네 가지다.

- `진행`: 필요한 확인을 마친 뒤 사용자가 연습상 진행을 선택
- `추가 확인`: 질문이나 서류 확인을 더 수행
- `보류`: 현재 결정을 미루고 확인 시간을 확보
- `중단`: 연습상 계약 진행을 멈춤

`해지`는 이미 체결된 계약을 전제로 오해할 수 있어 첫 계약 전 시나리오에서는 사용하지 않는다.

정상 종료 조건:

- 정해진 대화 턴을 완료하고 행동을 선택함
- 사용자가 `연습 종료`를 선택함
- 사용자가 중단을 선택함

비정상 종료 후에는 마지막 저장 상태에서 재개하거나 새로 시작할 수 있어야 한다. 완료된 세션은 내용을 변경하지 않고 다시 보기만 허용한다.

## 7. 결과 표현 규칙

최종 결과는 다음 순서로 보여준다.

1. 잘한 대응
2. 놓친 확인 신호
3. 다음에 사용할 수 있는 짧은 문장
4. 계약 전 확인할 문서·조건
5. 계약금 송금 전 행동
6. 연결된 공식 근거
7. 같은 상황 다시 연습

다음 표현은 사용하지 않는다.

- 안전 점수·위험 점수·사기 가능성 점수
- 안전한 계약·위험한 계약이라는 종합 결론
- 적법·위법 확정
- 계약 진행·거절 권고
- 검증된 계산식이 없는 예상 금전 피해액

사용자의 `진행` 선택은 사용자의 연습상 선택이다. 서비스가 계약 체결을 추천한 것으로 표현하지 않는다.

## 8. 시나리오 생성 원칙

- 첫 구현은 승인된 합성 시나리오 한 개로 시작한다.
- 이후 `무작위 상황`은 승인·버전 관리된 시나리오 카탈로그 안에서 선택한다.
- 런타임 LLM이 금액·당사자·권리관계·정답을 무제한으로 생성하지 않는다.
- 시나리오가 바뀌면 버전을 올리고 기존 연습 결과는 당시 버전에 고정한다.
- 실제 개인정보·실제 계약 문서·실제 상대방 영상은 사용하지 않는다.

## 9. AI·규칙·RAG 권한

- Python 계약 규칙 엔진: 합성 계약 내용의 `RuleStatus`와 `urgency`를 결정한다.
- 연습 행동 평가기: 사용자가 기대 행동을 수행했는지 별도로 평가한다.
- Gemini 답변 분류: 사용자 문장에서 행동 후보를 고정 JSON으로 구조화하되 계약 판정을 변경하지 않는다.
- RAG: 연결된 공식 근거만 검색하고 계약 판정이나 행동 수행 여부를 변경하지 않는다.
- Gemini 생성: 근거 기반 복기 문장과 후속 질문을 생성하되 규칙 결과를 변경하지 않는다.
- Guardrail: 단정 표현·근거 없는 조언·개인정보 노출을 차단한다.
- 로컬 7B: 첫 시뮬레이션의 크리티컬 패스에서 제외한다.

## 10. 영상·음성 원칙

- 캐릭터 기준 이미지는 Gemini로 생성한다.
- 실제 인물을 모사하지 않은 합성 캐릭터를 사용한다.
- 상황 설명·기본 대기·압박 대기·재촉·긍정·회피·전환을 짧은 사전 제작 클립으로 분리한다.
- 영상, 음성, 자막, 효과음은 가능한 한 분리해 수정 비용을 줄인다.
- PC와 모바일 자산을 각각 검수한다.
- image-to-video와 음성합성 제공자는 아직 `TODO`다.
- 제공자 후보와 품질 비교 기준은 [`practice-simulation-media-provider-research.md`](practice-simulation-media-provider-research.md)에 기록한다.
- 외부 호출·비용 발생·자산 생성은 사용자 승인 후 실행한다.

## 11. UX·접근성·안전 규칙

- 모든 화면에 `가상 연습`임을 표시한다.
- 언제든지 연습 중지·건너뛰기·음소거·자막 사용이 가능해야 한다.
- `prefers-reduced-motion`에서는 반복 동작을 줄이고 정지 이미지를 제공한다.
- 영상 재생 실패 시 Gemini 기준 이미지를 fallback으로 표시한다.
- 사용자의 입력 도중에는 압박 강도를 올리지 않는다.
- 카메라·마이크는 첫 버전에서 사용하지 않는다.
- 텍스트 답변은 연습 평가와 복기 목적에 필요한 범위만 저장한다.
- 타 사용자의 연습 세션과 결과에 접근할 수 없어야 한다.
