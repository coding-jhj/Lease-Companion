# PoC 범위

> PoC는 기술 파이프라인의 실현 가능성 검증에 초점을 둔다. MVP 사용자 기능(`mvp-scope.md`)과 혼용하지 않는다.

## 입력

- 계약서
- 등기사항증명서
- 건축물대장

## 처리 흐름

1. 계약서·등기사항증명서·건축물대장 입력
2. 멀티모달 LLM 핵심 정보 추출
3. 필드 정규화
4. Python 규칙 엔진 문서 간 비교
5. 공식 법령·가이드 RAG
6. 확인 항목·질문·체크리스트 생성

## 출력

- 확인 필요 항목
- 임대인·공인중개사 대상 질문
- 서명 전 체크리스트

## 검증 목표

- 멀티모달 추출이 핵심 필드를 구조화 스키마로 뽑아내는가
- 정규화가 주소·금액·날짜·이름을 비교 가능한 형태로 만드는가
- 규칙 엔진이 문서 간 불일치·누락을 분류하는가
- RAG가 공식 근거를 검색해 연결하는가
