# PRACTICE-BROKER-PRESSURE-001 — 즉시 계약 압박

- 시나리오 버전: `1.0.0`
- 데이터 유형: 전부 합성
- 계약 유형·단계: 전세 / 계약금·가계약금 입금 전 / 서명 전
- 상대 역할: 가상 공인중개사
- 난이도: 기본
- 작성 상태: 1단계 원본·정답표 작성 완료
- 검토 상태: 규칙·출처·fixture 자동 검증 및 사용자 내용 승인 완료(2026-07-20)

## 1. 학습 목표

사용자가 “다른 사람이 곧 계약한다”는 재촉을 받아도 즉시 동의하거나 송금하지 않고 다음 네 행동을 수행하도록 연습한다.

| 행동 ID | 기대 행동 | 완료 의미 |
|---|---|---|
| `PA01` | 즉시 결정 유예 | 필요한 확인이 끝난 뒤 결정하겠다고 명확히 말함 |
| `PA02` | 최신 등기사항증명서·소유자 확인 | 목적물, 최신 권리관계, 공동소유자와 계약 권한 확인을 요청함 |
| `PA03` | 특약·보증금 반환 조건 문서 확인 | 구두 설명이 아니라 반환 시점·조건과 특약 문구를 계약서에서 확인하자고 요청함 |
| `PA04` | 확인 전 계약금·가계약금 송금 보류 | 계좌 명의와 계약 상대를 확인하기 전에는 송금하지 않겠다고 명확히 말함 |

한 문장에 네 행동을 모두 요구하지 않는다. 전체 대화에서 각 행동을 한 번 이상 명확히 수행하면 인정한다.

## 2. 합성 계약 사실

| 항목 | 합성 값 |
|---|---|
| 목적물 | 서울특별시 가온구 나래로 24, 101동 804호 |
| 계약 유형 | 전세 |
| 보증금 | 260,000,000원 |
| 계약금 | 26,000,000원 |
| 잔금 | 234,000,000원 |
| 중개사가 요청한 가계약금 | 2,000,000원 |
| 계약 예정 임대인 | 윤서준 |
| 등기상 소유자 | 윤서준 1/2, 윤서연 1/2 |
| 가상 공인중개사 | 김하람 |
| 안내된 입금 명의 | 김하람(공인중개사) |
| 등기사항증명서 열람일 | 2026-07-10 |
| 근저당권 기재 | 있음, 채권최고액 120,000,000원 |
| 보증금 반환 초안 | “새 임차인 입주 후 협의하여 반환한다.” |

이 값들은 학습용 합성 사실이다. 주소의 `가온구`와 인물 이름은 실제 계약·인물을 가리키지 않는다. 계좌번호·연락처·주민등록번호는 생성하거나 저장하지 않는다.

## 4. 대화 원본

### 상황 설명

> 가상 연습입니다. 아직 계약서에 서명하지 않았고 계약금이나 가계약금도 보내지 않았습니다. 가상 공인중개사가 다른 방문 예정자를 이유로 오늘 결정을 요청합니다. 화면의 시간은 실제 계약 기한이 아닙니다.

### TURN-01 — 결정 유예

- 첫 질문: “조건은 설명드린 대로예요. 이 집으로 진행하시겠어요?”
- 0~5초 기본 대기: 조용히 사용자의 답변을 기다림
- 5~10초 압박 대기: 시계를 확인하고 계약서를 가볍게 정리함
- 10초 이후 재촉: “다른 분도 곧 보러 오기로 했어요. 지금 결정하셔야 합니다.”
- 적절한 확인: “네, 확인할 시간을 먼저 가지시는 게 좋겠습니다. 무엇부터 확인하시겠어요?”
- 부분 확인: “어떤 확인이 필요한지 한 가지만 더 구체적으로 말씀해 주세요.”
- 애매함·회피: “그럼 오늘 진행 여부만 먼저 정해 주실 수 있을까요?”
- 학습 행동: `PA01`

### TURN-02 — 등기·소유자 확인

- 질문: “등기사항증명서는 열흘 전에 확인해 두었고 임대인 한 분은 곧 오실 예정입니다. 이 정도면 괜찮으시죠?”
- 적절한 확인: “최신 등기와 공동소유자 확인을 요청하셨군요. 확인 자료를 준비하겠습니다.”
- 부분 확인: “등기에서 어떤 내용과 누구의 권한을 확인할지 더 말씀해 주세요.”
- 애매함·회피: “전에 확인한 자료니까 우선 계약하고 나중에 다시 보면 어떨까요?”
- 학습 행동: `PA02`

### TURN-03 — 특약·반환 조건 확인

- 질문: “보증금은 새 임차인이 들어오면 협의해서 돌려드리는 것으로 적어 두었습니다. 자세한 내용은 나중에 정리해도 됩니다.”
- 적절한 확인: “반환 시점과 조건을 계약서 문구로 명확히 하자는 요청이군요.”
- 부분 확인: “어떤 시점과 조건을 문서에 적을지 더 구체적으로 말씀해 주세요.”
- 애매함·회피: “보통 이렇게 진행하니 우선 서명부터 하셔도 됩니다.”
- 학습 행동: `PA03`

### TURN-04 — 송금 보류

- 질문: “그럼 김하람 명의 계좌로 가계약금 200만 원만 먼저 보내 주세요. 확인 자료는 송금 후 보내드리겠습니다.”
- 적절한 확인: “계좌 명의와 계약 권한을 확인한 뒤 송금하겠다는 말씀이군요.”
- 부분 확인: “계좌 명의 확인과 송금 보류 의사를 함께 말씀해 주세요.”
- 애매함·회피: “금액이 크지 않으니 자리만 잡아 두시죠.”
- 학습 행동: `PA04`

### 종료

> 연습 대화가 끝났습니다. 진행·추가 확인·보류·중단 중 선택한 행동과 대화에서 확인한 내용을 함께 복기합니다.

## 5. 답변 평가 정답표

| 상태 ID | 화면 상태 | 결정 기준 | 점수 처리 |
|---|---|---|---|
| `appropriate_check` | 적절한 확인 | 현재 턴의 필수 의미를 명확히 표현 | 대응 행동 인정 |
| `partial_check` | 부분 확인 | 확인 의도는 있으나 대상·조건·보류 중 일부가 없음 | 표현된 의미만 인정 |
| `ambiguous_answer` | 애매한 답변 | “생각해 볼게요”처럼 확인 행동이 특정되지 않음 | 행동 미인정, 재질문 |
| `avoidance` | 회피 | 무관한 답변, 즉시 동의, 확인 요청을 피함 | 행동 미인정, 짧은 재질문 |
| `no_response` | 미응답 | 사용자 입력이 없고 정해진 대기 시점 도달 | 오답 처리하지 않고 재촉 1회 |
| `needs_review` | 확인 필요 | provider 실패·timeout·형식 오류 또는 의미 충돌 | 오답 처리하지 않고 다시 답할 기회 |

금지 사항:

- 특정 키워드 하나만으로 행동을 인정하지 않는다.
- “계약하지 마세요”, “사기입니다”, “안전합니다” 같은 결론을 정답으로 요구하지 않는다.
- provider 실패를 사용자 오답이나 행동 누락으로 저장하지 않는다.
- 동일한 정규화 입력·턴에는 항상 동일한 기대 상태·행동·다음 턴을 사용한다.

구체적인 필수 의미·부분 인정 의미·고정 입력/출력 예시는 [`../../../data/sample/practice-scenarios/PRACTICE-BROKER-PRESSURE-001/answer-key.json`](../../../data/sample/practice-scenarios/PRACTICE-BROKER-PRESSURE-001/answer-key.json)에 둔다.
