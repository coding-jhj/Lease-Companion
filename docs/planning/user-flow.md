# 사용자 흐름 (기준 8단계)

프로젝트 전체가 따르는 기준 흐름이다. 기존 6단계 단일 세션 흐름을 회원·계약 건 관리가 포함된 8단계로 교체했다.

분석은 단일 세션으로 끝나지 않는다. 로그인한 사용자의 **계약 건(ContractProject) 단위**로 분석 결과·체크리스트·계약 직후 행동 상태를 저장하고 재조회한다.

## 1. 회원가입·로그인

- 회원가입 / 로그인 / 로그아웃 / 회원 탈퇴
- 인증 세부 기술은 미정(TODO). 회원 기능 자체는 MVP 확정 범위.

## 2. 계약 대시보드·계약 건 생성

- 사용자별 계약 건 목록 조회
- 계약 건 생성·삭제

## 3. 계약 상황 입력

- 계약 단계 선택
- 계약 상황(전세/보증부 월세/일반 월세 등) 입력

## 4. 계약서·등기 등 문서 업로드

- 계약서·특약: **필수**
- 등기사항증명서·중개대상물 확인설명서: **선택**
- 디지털 PDF 텍스트 추출(PyMuPDF·PDF.js) / 스캔 PDF·사진 OCR(상용 LLM Gemini 3.5 Flash VLM 통합)

## 5. 추출값 확인·수정

- 핵심 정보 구조화 후 사용자가 추출값을 확인·수정
- **분석(6단계) 전에 반드시 사용자 확인·수정 단계를 거친다.**

## 6. 상용 LLM 구조화·규칙 엔진·RAG·상용 LLM 분석

- 상용 LLM(Gemini 3.5 Flash): 조항 유형·명확성 후보 구조화 (선택: 로컬 7B 성능비교 실험 — MVP 크리티컬 패스 제외)
- Python 규칙 엔진: 문서 내부 판정과 문서 교차검증 (최종 판정)
- RAG: 공식 근거 검색
- 상용 LLM: 저신뢰 결과 재검토, 쉬운 설명·질문·행동 생성

## 7. 판정·원문 증거·공식 근거·질문·행동 리포트

- 4영역 12항목 판정 결과(상태 + 시급도)
- 원문 증거(근거 페이지·문구)
- 공식 근거(법령·공공자료 인용)
- 임대인·공인중개사 질문, 사용자 행동

## 8. 저장된 체크리스트·계약 직후 행동 관리

- 서명 전 체크리스트 상태 저장·재조회
- 계약 직후 권리 확보 행동 상태 저장·재조회
- 추출·판정·근거 오류 사용자 피드백

## 설계 원칙

- 추출값은 분석 전에 사용자가 확인·수정할 수 있어야 한다.
- 결과는 계약 건 단위로 저장하고 다시 열람할 수 있어야 한다.
- 판정 결과는 [공통 결과 상태 9개 + 시급도 5개](../data/judgment-spec.md)로 표현하며, `안전`·`위험`·`사기 가능성 점수` 같은 종합 판정은 사용하지 않는다.
