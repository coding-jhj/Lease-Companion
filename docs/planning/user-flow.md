# 사용자 흐름 (기준 8단계)

프로젝트 전체가 따르는 기준 흐름이다. 기존 6단계 단일 세션 흐름을 회원·계약 건 관리가 포함된 8단계로 교체했다.

분석은 단일 세션으로 끝나지 않는다. 로그인한 사용자의 **계약 건(ContractProject) 단위**로 분석 결과·체크리스트·계약 직후 행동 상태를 저장하고 재조회한다.

## 1. 회원가입·로그인

- 회원가입 / 로그인 / 로그아웃 / 회원 탈퇴
- 인증은 JWT Bearer(Python: PyJWT + Passlib-bcrypt)로 구현 완료(2026-07-16, → [`../decisions/2026-07-16-mvp-platform-stack.md`](../decisions/2026-07-16-mvp-platform-stack.md)). refresh token·운영 키 정책은 TODO.

## 2. 계약 대시보드·계약 건 생성

- 사용자별 계약 건 목록 조회
- 계약 건 생성·삭제

## 3. 계약 상황 입력

- 계약 단계 선택
- 계약 상황(전세/보증부 월세/일반 월세 등) 입력

## 4. 계약서·등기 등 문서 업로드

- 계약서·특약: **필수**
- 등기사항증명서·중개대상물 확인설명서: **선택**
- 디지털 PDF 텍스트 추출(PyMuPDF·PDF.js) / 스캔 PDF·사진 OCR(상용 LLM Gemini 3.5 Flash VLM 통합)

## 5. 추출값 확인·수정

- 핵심 정보 구조화 후 사용자가 추출값을 확인·수정
- **분석(6단계) 전에 반드시 사용자 확인·수정 단계를 거친다.**

## 6. 상용 LLM 구조화·규칙 엔진·RAG·상용 LLM 분석

- 상용 LLM(Gemini 3.5 Flash): 조항 유형·명확성 후보 구조화 (선택: 로컬 7B 성능비교 실험 — MVP 크리티컬 패스 제외)
- Python 규칙 엔진: 문서 내부 판정과 문서 교차검증 (최종 판정)
- 사용자 확인 특약: 결정론적 6유형 후보 매칭 → Python J10~J12 판정 → 특약 원문·R/J 상태 기반 공식 근거 Top-3 검색
- 상용 LLM: 저신뢰 결과 재검토, 근거 기반 쉬운 설명·확인 질문·수정 요청·행동 생성. 근거가 없으면 질문과 한계만 제공

## 7. 판정·피해 유형 비교·원문 증거·공식 근거·질문·행동 리포트

- 4영역 13항목 판정 결과(상태 + 시급도)
- 원문 증거(근거 페이지·문구)
- 공식 근거(법령·공공자료 인용)
- 임대인·공인중개사 질문, 사용자 행동
- 제출 자료 범위를 명시한 주요 금전피해 유형 비교
- 수정·추가 요청 문구와 계약 전·계약 중·잔금·입주 당일·계약 후 행동
- 검증된 유사 참고 사례(공식 근거와 분리, 없으면 빈 상태)
- `확인이 필요한 특약` 카드와 전체 리포트 PDF(특약 원문·Python 판정·공식 근거·질문·수정 요청·한계)

## 8. 저장된 체크리스트·계약 직후 행동 관리

- 서명 전 체크리스트 상태 저장·재조회
- 계약 직후 권리 확보 행동 상태 저장·재조회
- 추출·판정·근거 오류 사용자 피드백

## 별도 확장 흐름: 계약 연습 시뮬레이션

> 상태: 2026-07-22 승인된 합성 시나리오 3개가 AI 평가·Backend 세션 API·Frontend 텍스트 대화·결과 저장과 연결됐다. 실제 아바타 미디어 연결은 후속이다. 위 기준 8단계와 실제 계약 건 이력을 변경하지 않는다.

1. 로그인 후 `/choose-mode`에서 `계약 연습 시뮬레이션` 또는 `실전 계약 점검` 선택
2. 승인된 합성 계약 상황 안내
3. 정답 표시 없이 모의 계약서 요약·특약 확인
4. 가상 임대인·공인중개사의 대사와 계약서를 함께 보며 텍스트로 질문·요청
5. 사용자 발화에 따른 후속 반응과 서명 직전 압박 대화
6. 진행·특약 수정 요구·보류 중 최종 행동 선택
7. 실제 사용자 발화·확인한 행동·놓친 확인·추천 문장 복기 또는 다시 시작

canonical 진입 라우트는 `/practice`다. 세 시나리오는 실제 계약 분석 결과와 분리된 합성 fixture를 사용하며, Backend 세션 API가 턴과 최종 복기를 사용자별로 저장·재조회한다. 기존 `/practice/signing`은 첫 시나리오 소개 화면으로 이동하는 호환 경로다.

상세: [`practice-simulation-product-rules.md`](practice-simulation-product-rules.md)

## 설계 원칙

- 추출값은 분석 전에 사용자가 확인·수정할 수 있어야 한다.
- 결과는 계약 건 단위로 저장하고 다시 열람할 수 있어야 한다.
- 판정 결과는 [공통 결과 상태 9개 + 시급도 5개](../data/judgment-spec.md)로 표현하며, `안전`·`위험`·`사기 가능성 점수` 같은 종합 판정은 사용하지 않는다.
