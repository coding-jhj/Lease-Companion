# normalization/

## 책임

추출된 원문 값을 규칙 엔진이 비교할 수 있는 정규 형식으로 변환한다. 이름·주소·금액·날짜를 표준화하되, **원문 값은 보존하고 정규화 값을 별도로 남긴다.** 판정은 하지 않는다.

## 구현 파일

- `core.py` — 이름 공백 제거와 주소 비교용 기본 정규화
- 금액·날짜 parsing과 숫자-한글 금액 비교는 추출·규칙 계층에서 canonical 값을 보존한 채 수행한다.

## 입력

- `extraction/` 출력: 문서별 원문 필드 값

## 출력

- 원문 값 + 정규화 값 쌍 (교차검증·표기 일치 판정용)
- 정규화 실패·모호 값은 플래그로 표시
