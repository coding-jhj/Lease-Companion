# rules/

## 책임

Python 규칙 엔진. 문서 내부 값 판정과 문서 간 교차검증의 **명시적 최종 판정**을 담당한다. LLM·로컬 모델은 이 결과를 임의로 변경하지 않는다. 판정마다 결과 상태(9개)와 시급도(5개)를 부여한다.

## 하위 구조

- `field_rules/` — 문서 내부 필드 판정 (금액 기재 상태 J06, 숫자-한글 일치 J07, 일자 일관성 J08, 관리비 명확성 J09 등)
- `cross_document/` — 계약서-등기사항증명서 등 문서 간 교차검증 (임대인=소유자 J01, 주소 일치 J02, 계좌 명의 J05 등)
- `urgency/` — 판정 결과에 시급도(`즉시 확인`·`계약 전 확인`·`계약 직후 조치`·`참고`·`분석 불가`) 부여

## 입력

- `normalization/` 정규화 값, `classification/` 조항 후보, 문서별 필드 값

## 출력

- 판정별 결과 상태(`일치`·`불일치`·`명확`·`불명확`·`미기재`·`상충 가능`·`확인 필요`·`확인 불가`·`적용 제외`) + 근거 필드 + 시급도
- **`안전`/`위험`/사기 점수 같은 종합 판정은 내지 않는다.**
