# ai/AGENTS.md

`ai/` 전용 지시서. 루트 [`../AGENTS.md`](../AGENTS.md)를 전제로 하며, 여기서는 AI 모듈 고유 규칙만 정의한다.

## 컴포넌트 역할

- 멀티모달 LLM: 문서에서 핵심 정보 **추출** 담당.
- Python 규칙 엔진: 문서 비교·확인 필요 항목 **분류** 담당.
- RAG: 공식 근거 **검색·설명** 담당.
- 생성 모듈: 질문·체크리스트·행동·요약 **생성** 담당.

## 규칙

- LLM은 규칙 엔진의 결과를 임의로 변경하지 않는다.
- **추출값**(문서에서 읽은 값)과 **생성값**(모델이 만든 질문·요약)을 항상 구분한다.
- 모든 AI 결과는 구조화된 스키마(`src/lease_companion_ai/schemas/`)로 반환한다.
- 근거가 부족하면 추측하지 않고 `정보 부족` 또는 `확인 필요` 상태로 반환한다.
- 프롬프트를 코드에 길게 하드코딩하지 않는다. 원본은 `prompts/`에 두고 버전을 관리한다.
- 프롬프트 변경 시 버전과 평가 결과를 확인한다. (`docs/ai/prompt-management.md`)
- guardrails 모듈은 단정적 표현(가능·안전·사기·합법 판정)과 잘못된 출력을 제한한다.

## 모듈 경계

| 모듈 | 책임 |
|------|------|
| `extraction/` | 문서에서 핵심 필드 추출 |
| `normalization/` | 추출값 정규화(주소·금액·날짜·이름) |
| `rules/` | 문서 비교·확인 필요 항목 분류 |
| `rag/` | 공식 근거 검색 |
| `generation/` | 질문·체크리스트·행동·요약 생성 |
| `guardrails/` | 단정 표현·잘못된 출력 제한 |
| `pipelines/` | PoC·MVP 처리 흐름 연결 |
| `schemas/` | AI 입출력 구조 |
| `prompts/` | 프롬프트 원본·버전 |

## 테스트

- 추출·정규화·규칙·검색·생성·전체 흐름 테스트를 `tests/` 하위에서 분리한다.
- 테스트하지 않은 내용을 통과했다고 쓰지 않는다.
